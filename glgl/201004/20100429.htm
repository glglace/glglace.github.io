20100429-- 嘰哩呱啦ACE--第744回--再生龍 Clonezilla Live -4
  • 早上六點出頭就被唸小一的女兒挖起床來，因為今天校外教學！我問女兒說你該不會太興奮吧！她說：哪有！我只是睡不著~ 接著催促我趕快出門，結果還沒七點就到學校了。連續幾天陰雨，今天總算放晴了，相信她應該會玩得很開心，看著女兒飛快的成長，似乎就只看到她一直往前衝的 背後留下的塵煙，細細品嚐自己的父母親看我成長的滋味...
  • 今天一早到學校，繼續還原電腦教室裡的學生電腦，昨天下午花了兩小時還原了6臺，今天要把剩下的 12 臺還原好。

  • 昨晚在噗浪上噗了昨天第743回的出刊通知 後，阿剛老師問說：「為什麼不直接把你做好的再生龍備份檔，用drbl去一次群播還原就好了，一次搞定所有電腦！」我說：「因為我不會用drbl」。

  • 在我印象中，企鵝龍 DRBL (Diskless Remote Boot in Linux) 是用來建置無硬碟電腦教室的，Server 提供 Client 一切軟體。跟再生龍 Clonezilla Live 一樣都是由國家高速網路與計算中心所開發。

  • 阿剛又說：「我看著簡 易電腦教室自動還原系統操作實例--使用DRBL-Live--教學PDF做，一下子就 把全電腦教室還原了！」

  • 我昨晚估計，今天早上大概要花四個小時才能把剩下的 12 臺還原好，雖然很費時，但至少是可以成功的。不過阿剛的一下子實在很吸引我，於是我就大略的看了一遍，似懂非懂。想說那麼來模擬演練一遍好了，果然模 擬演練過後就懂了！主要步驟為：
    1. 取得DRBL-LiveCD iso file 並燒錄成光碟片
    2. 安裝好一臺學生電腦作業系統和上課要用到的軟體，再加裝 winroll以便大量佈署學生電腦主機的電腦 名稱、工作群組、IP
    3. 在電腦教室的教師電腦上，使用DRBL-LiveCD 來接收學生範本主機硬碟/分割區的映像檔 (image file)。
    4. 在電腦教室的教師電腦上，使用DRBL-Live CD 透過群播 (multicast) 派送學生範本主機硬碟/分割區的映像檔 (image file) 給其他學生電腦。
  • 看著教 學PDF演練的過程中，在第15頁的「是否配置SSHD」選Y 結果我家裡的舊電腦卡在這裡很久，放棄，重做一遍時這裏選 N 才過關。昨晚花了40分鐘把65頁的檔案看玩並演練過後，有信心明天去學校應該可以一個小時內還原搞定12臺學 生電腦！

  • 今早到學校後，一個五年級一個四年級的學生來電腦教室幫我，希望能在他們校外教學出發之前讓他們能看到群播的盛況！


  • 7:28 請學生幫忙改 BIOS設定，First Boot Device改為「Lan」，我順便解釋給學生聽，學生看到螢幕上都是英文，而他又很想學電腦裝修，馬上就很有感覺的跟我說，老師我要好好學英文了！聽了 真的很欣慰，下週開始來幫他做英語補救教學，我希望是先敎他一次，再由他去帶中年級需要補救英語的幾個孩子。

  • 7:29 在電腦教室的教師電腦上，使用DRBL-LiveCD開機，由於我昨天就已經做好了學生範本主機硬碟/分割區的映像檔 (image file)，所以直接從教 學PDF的第19頁開始做起。
學生電腦的IP、電腦名稱、使用者名稱
  • 7:31 開機中，7:33 拍了右上圖，正好是做錯的地方，應該要選 static


  • 7:48 教師電腦上，使用DRBL-LivCD開機、設定就緒


  • 7:49 學生電腦開機準備接收映像檔，但失敗，學生很失望，沒有看到我形容的咻咻咻地發射群播的畫面，我說不怕做錯，只怕找不出哪裡錯，我查一下教 學PDF發現第39頁BIOS設定的 Onboard Lan Boot Rom 改為「Enabled」漏了做，再請學生補做。


  • 8:07 總算學生電腦可以接收到老師電腦的DRBL-Live 所派送的還原指令！


  • 等了好一會怎麼都沒有開始發射群播，懷疑哪邊出狀況，正和學生討論時，8:09忽然12臺電腦的畫面刷刷的閃過！我和學生都嚇了一跳！三個人高興得又叫又跳，我們成功了！而且是在校外教學上車前！

  • 高興一下而已，我發現傳送速度一開始有4百多M/min，一分鐘左右以後，掉到一百多M/min，這樣要將近一小時才能群播完成，不曉得為什麼 ？

  • 9:42  群播還原完成10臺，也改好個別學生電腦。另有2臺學生電腦開機時來不及而落單，再群播給這兩臺，這次我有拔掉對外網路線，但傳送速度還是隻有一百多M/min。

  • 這次還原還有一個意外收穫，原本好幾臺學生電腦的音效怎麼轉都很小聲，以為是 onboard 音效晶片故障，這次還原之後，竟然都變大聲了！

  • 還原之後，還需要手動改學生電腦的IP、電腦名稱、使用者名稱，因為我昨天做映像檔時還不知道有 winroll 。還好，一臺只要一分鐘就改好。

  • 雖然不是像昨天幻想的只要一小時就可以群播還原，不過把這個技術學會了，下次再做應該就會更省時間了。 Oh, No! 最好是不要有下次了...
