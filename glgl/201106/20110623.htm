20110623--嘰哩呱啦ACE--第994回--自由的風-39-救回被刪除的資料
  • 收到一份公文說暑假前要列印給家長一封信-關心孩子的網路世界，可是公文上所指引的eteacher 網站上的PDF檔，很不適合列印，印成A4字太小，印成 A3 又太大張不好發放，我想要將之印成A4正反兩面，去影印機那邊玩不出來怎麼印，想說google看看有沒有本來就是 A4正反兩面的PDF，還真的有！
  • 6/21 下午我在學校的windows電腦上把數位相機記憶卡中的照片傳到我的行動碟時，傳完後沒幾秒就響起了放學的鐘聲，聽到鐘聲就想起老婆交代要去幼稚園接兒子回家，一急之下就馬上把行 動碟拔出來，啊！慘了拔太快沒有安全退出，趕快再插回去檢查一下，嗚嗚~照片全部不見了！
    　
  • 趕快發一個噗問噗友看 看有沒有救？然後把電腦關機下班去接兒子，16:50回到家後趕快上網查一下，在ubuntu 中有沒有救回隨身碟內刪除資料的方法？查一了會兒沒什麼收穫。
    　
  • 晚上７點多忙晚家事後，用「搶救資料 linux」當關鍵字，找到 五種最好且免費的檔案救援軟體，但第 一個 testdisk 看其說明我以為只是用來救消失的磁碟分割區，又看了一些網頁都沒有收穫，有點灰心，但還是不想放棄！我會遇到的問題，自由軟體社 群的人一定也曾經遇到，我相信會有好心人寫一個這種用途的自由軟體的，找不到可能是我關鍵字下得不好。
    　
  • 我想到以前想救回數位相機誤刪資料時，阿剛老師曾推薦的recuva，所以我就改用「recuva ubuntu」當關鍵字，想看看 linux 下有沒有類似 recuva 的自由軟體，結果找到一篇How to recover deleted files with Photorec - Ubuntu，裡面有影片可以看。

  • 雖然很心急但還是耐著性子看，看了錄影之後才知道原來我下午找到的那篇文章裏面提到的 testdisk，裏面竟然還有一個 photorec 可以做類似recuva 的事。
    　
  • 我想我運氣真的很不錯，請看上圖，該網頁中真的有出現 ubuntu 和 recuva 兩個字詞。
    　
  • 影片看了一半我就迫不及待的開始動手了！首先要安裝 testdisk
    • sudo apt-get install testdisk
      　
  • 安裝好之後 sudo photorec，然後一步一步選擇符合自己的行動碟狀況的選項，就開始救援了！詳細步驟請看影片。
    　
  • 從上圖中可到我一共救回了多少檔案，但救回的檔案所在的資料夾，是屬於root的，這樣一般使用者無法做存取與改 變。比較簡單的方法是，對資料夾按右鍵，選「以系統管理員的身份開啟」，然後再將之放回fat32格式的行動碟中，權限自然就消失啦。
  • 看到照片檔都回來了真是開心呀！不過我並沒有要全部已刪除的都救回來呀！所以我只選擇 6/21 拍的照片來救。
    　
  • 但是一看照片檔的詳細資料才發現，糟糕！全部都是救回的日期與時間，還好我之前有寫一個命令稿，可以把檔案的時間戳 記改回成exif上紀錄的拍照時間，按個右鍵就輕輕鬆鬆的，不到十秒鐘就把七百多張照片的日期時間都改好了。
    　
  • 既然花了那麼多時間解決這個問題，那 更要再花點時間將自己的經驗分享出來，當別人遇到類似問題在網路上尋找解決辦法時，也許就會找到我這篇網誌，人人為我，我為人人，就是這個道理。
