  • 轉換作業系統，必須要能做在原系統上能做的所有事。因 為有這樣的想法，所以以前雖然曾經嘗試過好幾次想要「棄 Windows 投 Linux」，可是總是遇到問題卡住就放棄了。這一次的嘗試，希望在噗友群的協助下能夠成功。何謂 成功？我希望90%以上使用電腦時，都是在自由的作業系統下使用自由軟體和免費軟體。
    　
  • 把一堆數位照片縮小，是我經常在做的事。我目前使用的相機 SONY TX7，雖可拍千萬畫素，但我沒那個需求，我都設定五百萬畫素拍 2592x1944，但一張照片存檔要 2~3MB我覺得還是太佔硬碟空間了。
    　
  • 我一個月大約會拍800~1000張照片左右，如果都不縮小，大約要用掉 2.5G，那麼一年就要 30G的硬碟空間來儲存。由於我對影像品質並沒有太多要求，套句阿亮笑長的話：「我的耳朵和眼睛都沒有很高級」，所以我習慣把數位相片傳輸到硬碟後，就立 刻將之縮小到 2400x1800，在 Windows下我是這 樣做的：
    　


  • Microsoft Windows XP 的 PowerToys 裡 面的 image resizer，安裝好了之後，在檔案 總管中只要選取一張或多張照片，然後按右鍵，選擇Resize Pictures， 會有四種預設的大小可以選擇，如果都沒有中意的，可以按 Advanced 按鈕自訂影像大小，如此就可以輕鬆的轉換影像大小。
    　
  • 如此一來，我一個月拍照的存檔量，大約是400MB。
    　
  • 由於這個動作我已經做了五年以上了，所以換到 Ubuntu，還是想繼續這麼做，於是我去檔案瀏覽器(相當於windows的檔案總管)，對著照片按右鍵試試看，很驚喜的發現居然有耶！也是可以一次選一堆照片一口氣縮小，真是太棒了！
    　
  • 不過我印象中之前用 ezgo8似乎沒有這個功能，於是我發了一噗問阿剛老師：在檔案瀏覽器中對著照片按右鍵有 resize 的功能，是您新增上去的，還是 ubuntu 原本就有的功能？
    　
  • 阿 剛老師說是他加上去的，我說：那太好了，請教來源，因為我想修改jpg 的壓縮比，但目前該軟體內的選項沒有。我發現一張2592x1944數位照片原本 2.5MB， resize 到 2400x1800之後卻還有 1.9MB，我預期應該是要壓縮到 1MB以下的。
    　
  • 阿剛回我說他在20100817有寫一篇網誌 Ubuntu 主題教室-快速將大量圖片縮小尺寸(nautilus-image-converter)，我從阿剛老師的網誌文章中，得知了關鍵字是 nautilus-image-converter，去google一下找到一篇 修 改 nautilus-image-converter 預設的縮圖尺寸，該部落格的名稱和副標題我很喜歡，「 有靠背金交椅--為了追求軟體自由而使用自由軟體的電腦門外漢的雜七雜八筆記與想法堆棧」。
    　
  • 但該文裡並沒有我想要的答案，不過我從該文內提到該文之前有一篇是只用右鍵選單，快速旋轉 縮放圖檔！（正體中文介面），於是照著文內的方法試試看。
    　

  • 方法簡述如下：
    1. 到這裡下載 nautilus-image-converter.mo
    2. 放到 /usr/share/locale/zh_TW/LC_MESSAGES 裡面即可
      　
  • 依樣畫葫蘆成功！
